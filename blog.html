1. localStorage vs sessionStorage

The main and only difference between localStorage and sessionStorage the time of expire. In localStorage data is not cleared until we delete it but in sessionStorage data is cleared automatically after finishing the session.
If we want to share data among multiple windows we should use localStorage. However, if we don't want to do so than we can use sessionStorage.



2. Global scope vs block scope

In global scope we can access the variable from everywhere. However, if we declare variable inside {} block we can get access only inside the block. Variable declared by var will not follow the block scope but let and const will follow. Block scope also called a sub-type of local scope. It is asserted that to use global variable is not good practice when they are not needed as all block variable can get access.



3. What is the event loop in javaScript?

The event loop is described to hold a runtime model for JavaScript's asynchronous programming which is not similar for C and Java. In other words we can say an event loop pulls stuff out of the queue and keeps onto the function call stack one after another whenever the call stack is empty.



4. Describe how many ways we get undefined?

undefined is a property of the global object. That is, it is a variable in global scope.
JavaScript uses the undefined value in the following situations:

1) The variable will have a value of undefined if it was uninitialized.

2) If you access a non-existing property of an object, you’ll get undefined.

3) When we don’t pass all the arguments at the time of calling the function, the parameters inside the function becomes undefined.

4) If we do not return a value it implicitly returns undefined.
